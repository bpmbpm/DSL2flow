# Руководство пользователя DSL2flow

## Описание

**DSL2flow** — браузерное приложение для моделирования и симуляции BPMN-процессов с использованием текстового языка DSL2 (Domain-Specific Language 2). Приложение позволяет описать бизнес-процесс в виде текста, автоматически сгенерировать EPC2-диаграмму, таблицу состояний и исполняемый JavaScript, а затем интерактивно проиграть процесс по ролям.

---

## Интерфейс приложения

### Заголовок

- **Кнопка EN / RU** — переключение языка интерфейса между английским и русским.

### Левая колонка

| Панель | Описание |
|---|---|
| **Загрузить пример** | Выбор и загрузка готового примера DSL2 |
| **Активная роль** | Выбор роли, от имени которой выполняются действия |
| **Статус процесса** | Текущая функция, роль, токен workflow, шаг, состояния документов |
| **Действия** | Кнопки для выполнения текущей функции процесса |
| **Управление** | Запуск/перезапуск процесса, экспорт таблицы в Excel |

### Правая колонка (вкладки)

| Вкладка | Описание |
|---|---|
| **EPC2 Диаграмма** | Автоматически сгенерированная схема процесса (нотация EPC2, Mermaid) |
| **Таблица состояний** | Трассировка выполнения: каждый шаг с токенами и переходами документов |
| **DSL Source** | Редактируемый исходный код DSL2; кнопка для перетрансляции |
| **Сгенерированный JS** | JavaScript-код движка, автоматически сгенерированный из DSL2 |

### Нижняя панель — Статус / Лог

- Журнал всех действий: инициализация, запуск, срабатывания событий, ошибки.
- **Сохранить лог** — скачивает лог в виде `.txt`-файла через стандартный диалог браузера.
- **Очистить** — очищает содержимое лога.
- Панель **растягивается** по высоте — потяните за правый нижний угол.

## Загрузка примера

1. В панели **«Загрузить пример»** выберите файл из выпадающего списка:
   - `example_en.dsl2` — пример на английском языке (согласование заявления на отпуск)
   - `example_ru.dsl2` — тот же процесс на русском языке
2. Нажмите кнопку **«Загрузить»**.
3. DSL-код появится на вкладке **DSL Source**, автоматически будет построена диаграмма.
4. Нажмите **«▶ Запустить / Перезапустить»** для начала симуляции.

---

## Основные действия

### 1. Запуск процесса

1. Загрузите пример или введите собственный DSL2 на вкладке **DSL Source**.
2. В панели **«Активная роль»** выберите роль (или «Supervisor» / «Супервайзер» для выполнения любых функций).
3. Нажмите **«▶ Запустить / Перезапустить»**.

### 2. Выполнение шагов

- Панель **«Действия»** показывает доступные действия для текущего шага.
- Нажмите кнопку действия (например, «Согласовать / Approve» или «Отклонить / Reject»).
- Системные функции (`role: System`) выполняются автоматически с задержкой 0,6 с.
- Смените роль в выпадающем списке, если текущий шаг требует другой роли.

### 3. Просмотр трассировки

- Перейдите на вкладку **«Таблица состояний»** для просмотра полной трассировки выполнения.
- Кнопка **«⬇ Excel»** экспортирует таблицу в файл `.xlsx`.

### 4. Редактирование DSL

1. Перейдите на вкладку **DSL Source** и отредактируйте код.
2. Нажмите **«⟳ Перетранслировать и перезапустить»** — код будет перекомпилирован и процесс начнётся заново.

## Синтаксис DSL2

```
process <НазваниеПроцесса>:
  title: "<Заголовок>"

  documents:
    <НазваниеДокумента>:
      states:
        - состояние1
        - состояние2
        - ...
      transitions:
        состояние1 --> состояние2 --> состояние3
        состояние2 --> отклонено

  roles:
    - Роль1
    - Роль2
    - ...

  workflow:
    start: <НачальноеСобытие>

    steps:
      - function <НазваниеФункции>:
          role: <Роль>
          system: <СистемаИсполнения>
          input_doc:
            - <Документ>.<состояние>
          output_doc:
            - <Документ>.<состояние>
          on_complete:
            - event: <НазваниеСобытия>
              condition: "decision == 'approved'"  # необязательно
              output_doc: <Документ>.<состояние>   # необязательно
              next: <СледующаяФункция | end>

    end: <КонечноеСобытие>
```

### Замечания по переходам документов

В секции `documents` рекомендуется явно указывать все возможные переходы состояний документа через нотацию mermaid:

```
transitions:
  шаблон --> подано --> согласовано_менеджером --> согласовано_директором
  согласовано_менеджером --> отклонено
  согласовано_директором --> отклонено
```

Это позволяет видеть, что в состояние `отклонено` можно попасть из двух разных состояний.

## Пример процесса: Согласование заявления на отпуск

**Участники:**
- `Сотрудник` — заполняет и подаёт заявление
- `Менеджер` — согласует или отклоняет
- `Директор` — финальное согласование
- `System` — отправляет уведомления

**Маршруты:**
1. Сотрудник подаёт → Менеджер согласует → Директор согласует → Уведомление об одобрении
2. Сотрудник подаёт → Менеджер отклоняет → Уведомление об отказе
3. Сотрудник подаёт → Менеджер согласует → Директор отклоняет → Уведомление об отказе
